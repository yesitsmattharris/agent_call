# Agent Call - Voice Server

## Gotchas

- **Never use bare `npx tsc` in Docker or CI.** It resolves to the wrong npm package (`tsc@2.0.4`, not TypeScript). Use `npx tsc --build` to force project-local resolution, or call `./node_modules/.bin/tsc` directly.
- **Use `process.env` with fallback in `prisma.config.ts`, not Prisma's `env()`.** `env()` throws if the variable is unset, breaking `prisma generate` during `npm install` / Docker builds / CI where `DIRECT_URL` isn't available. Use `process.env.DIRECT_URL ?? "postgres://localhost:5432/agent_call"` instead.
- **Dockerfile must use `npm ci --ignore-scripts`.** The postinstall script runs `prisma generate`, which needs the schema and config files. Those aren't copied until after `npm ci`. Use `--ignore-scripts` during install, then run `prisma generate` explicitly after copying the files.
- **`TwilioRealtimeTransportLayer` only listens to the Twilio WebSocket after `connect()` is called.** The message listener is registered inside `connect()`, not in the constructor. If you need to capture Twilio events (like `start`) before creating a session, listen on the raw socket directly.
- **Transport must see the Twilio `start` event to send audio back.** The transport captures `streamSid` from the `start` event and uses it for all outgoing audio (`media`, `mark`, `clear`). If the transport's listener misses the `start` event (because `connect()` hadn't been called yet), `streamSid` stays `null` and Twilio silently drops all outgoing audio, resulting in a silent call. Fix: save the raw start message and re-emit it on the socket after `connect()` resolves (`socket.emit("message", savedData, false)` triggers both `on` and `addEventListener` handlers in the `ws` library).
- **Pass caller info as TwiML `<Parameter>` elements, not from `customParameters` alone.** Twilio's `<Stream>` does not automatically forward webhook fields (From, To) to the WebSocket start event. You must explicitly add `<Parameter name="From" value="..." />` inside the `<Stream>` tag for them to appear in `start.customParameters`.
